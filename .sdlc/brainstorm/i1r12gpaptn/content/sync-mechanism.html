Sync Mechanism Design

Hybrid sync: local-first with cloud backup for pro users

1. When Sync Triggers

TriggerAction
Page load (pro user)Auto-scan local records, upload missing ones to cloud
New TTS generated (pro user)Save local + queue cloud upload
Manual "Upload to Cloud"Upload single record
Manual "Sync All"Force re-sync all local records

2. Sync Algorithm (useLibrarySync)

function useLibrarySync() { 1. Check subscription_tier === 'pro' 2. Read all local records from IndexedDB 3. Fetch existing cloud records from GET /api/library 4. Compute diff: records in local but NOT in cloud 5. If diff.length > 0: a. POST /api/library/sync with diff b. On success: update local records' sync_status.cloud = true c. Update UI via React Query invalidation 6. Show progress bar during sync 7. On error: show toast, retry option }

3. Error Handling & Edge Cases

Handled Cases

  • Network error during sync → show retry button, don't lose data
  • Quota exceeded → show warning + upgrade prompt
  • Duplicate records → idempotent (match by source_id)
  • Record deleted locally but exists on cloud → no action

Non-Goals (future)

  • Two-way merge conflicts
  • Real-time sync via WebSocket
  • Automatic retry with backoff

4. Sync Status Transitions

local=false, cloud=false → not applicable (no record) local=true, cloud=false → "Local" (green badge, needs upload) local=false, cloud=true → "Cloud" (orange badge, downloaded/viewed) local=true, cloud=true → "Synced" (blue badge, fully backed up) Transitions: Local → Synced (after successful sync) Synced → Local (if cloud delete, local remains) Cloud → Synced (if user downloads/plays, creates local copy) Local → (deleted) (delete in Local tab) Cloud → (deleted) (delete in Cloud tab) Synced → Local (delete in Local tab, cloud remains) Synced → Cloud (delete in Cloud tab, local remains)
