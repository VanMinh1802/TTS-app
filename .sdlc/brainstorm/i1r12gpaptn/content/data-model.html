Unified LibraryRecord Type

Flow by User Type

Free User

  1. Studio generates TTS → saves to IndexedDB (local only)
  2. Library page loads → reads from IndexedDB via useLibraryRecords
  3. Cloud tab visible but disabled with "PRO" badge
  4. CRUD: all on IndexedDB

Pro User

  1. Studio generates TTS → saves to IndexedDB (local)
  2. Library page loads → useLibraryRecords fetches cloud records
  3. useLibrarySync runs background sync: local → cloud
  4. Old records also synced on first load
  5. Each record shows correct status badge

Key Changes to Backend

New endpoint: POST /api/library/sync Request: { records: [{ id, text_content, voice_id, audio_data (base64), file_size_bytes }] } Response: { synced: [{ id, file_url, synced_at }], failed: [{ id, error }] } Modified: POST /api/library/upload Now also returns duration field Modified: GET /api/library Now returns records with status DB change: audio_records table ADD COLUMN duration FLOAT (nullable)
